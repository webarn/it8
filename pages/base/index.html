# typeof 和 instanceof的区别

typeof

typeof可以判断数据的数据类型,返回值有:string、boolean、number、function、object、undefined、symbol

instanceof

instanceof返回的是一个 布尔值 instanceof 不能判断基本数据类型

# undefined 和 null 的区别

首先 undefined 和 null 都是基本数据类型，这两个基本数据类型分别都只有一个值， 就是 undefined 和 null。undefined 代表的含义是未定义，null 代表的含义是空对象。一 般变量声明了但还没有定义的时候会返回 undefined，null 主要用于赋值给一些可能会返回对 象的变量，作为初始化。undefined 在 js 中不是一个保留字，这意味着我们可以使用 undefined 来作为一个变量名，这样的做法是非常危险的，它会影响我们对 undefined 值的 判断。但是我们可以通过一些方法获得安全的 undefined 值，比如说 void 0。当我们对两种 类型使用 typeof 进行判断的时候，Null 类型化会返回 object，这是一个历史遗留的问题。 当我们使用双等号对两种类型的值进行比较时会返回 true，使用三个等号时会返回 false。

# isNaN 和 Number.isNaN 函数的区别?

函数 isNaN 接收参数后，会尝试将这个参数转换为数值，任何不能被转换为数值的的值都会返 回 true，因此非数字值传入也会返回 true ，会影响 NaN 的判断。

函数 Number.isNaN 会首先判断传入参数是否为数字，如果是数字再继续判断是否为 NaN ，这种方法对于 NaN 的判断更为准确。

# 作用域和作用域链

作用域：

作用域可以理解为一个权限规则，决定了代码区块中变量和其他资源的可见性。

  • 全局作用域
  • 函数作用域
  • 块级作用域

作用域链

作用域链： 当 JavaScript 使用一个变量时， 会优先从变量的当前作用域下查找变量， 如果没有找到，再到它的上层作用域寻找，以此类推直到找到该变量或是已经到了全局作用域

# 闭包

闭包是指有权访问另一个函数作用域中的变量的函数 ——《JavaScript 高级程序设计》

用途:

1. 能够访问函数定义时所在的词法作用域(阻止其被回收)
2. 私有化变量
3. 模拟块级作用域
4. 创建模块

缺点:

1. 会导致函数的变量一直保存在内存中，过多的闭包可能会导致内存泄漏

# 原型和原型链

原型: js 中对象是一等公民, 除了null undefined皆为对象, 而 JavaScript 对象都有一个属性名为__proto__的公共属性,这个属性是对另一个对象的引用, 指向对象的prototype原型属性.

原型链: 在 Javascript 中如果访问一个对象本身不存在的属性或是方法，就首先在它的原型对象上去寻找，如果原型对象上也不存在，就继续在原型对象的原型对象上去寻找.所有对象的原型尽头是Object.prototype,Object.prototype的__proto__最终指向null

prototype 是函数才有的属性

instanceof就是根据原型来实现的 查看 instanceof 实现原理

# this 指向

this对象是是执行上下文中的一个属性，它指向最后一次调用这个方法的对象，在全局函数中，this等于window，而当函数被作为某个对象调用时，this等于那个对象。

# 1. 默认绑定(函数调用)

当一个函数不是一个对象的属性时，直接作为函数来调用时，this指向全局对象

var name = 'it8';

function fn1() {
  var name = 'it8-fn';
  console.log(this.name); // it8
}

fn1();

# 继承方式

# 1. 原型链继承

//父实例
function Person(name) {
  this.name = name;
  this.play = [1, 2, 3];
  this.setName = function () {
    console.log('setName: ', this.name);
  };
}

//子实例
function Student(age) {
  this.age = age;
  this.setAge = function () {
    console.log('setAge: ', this.age);
  };
}

const p = new Person('it8');

Student.prototype = p; // 子类型的原型为父类型的一个实例对象

var s = new Student(18); // 这里不能向父实例传参

p.name = 'it8-change'; // 修改父实例, 所有的继承的子实例的值也都会变化

s.play.push(4); // 子实例修改会直接父实例的值

console.log(p.name); // it8-change
console.log(s.name); // it8-change
console.log(p.play); // [1, 2, 3, 4]
console.log(s.play); // [1, 2, 3, 4]

// 子实例的constructor 指向父实例

# 2. 借用构造函数继承

这种方式关键在于:在子类型构造函数中用 call()调用父类型构造函数
















 












//父实例
function Person(name) {
  this.name = name;
  this.play = [1, 2, 3];
  this.setName = function () {
    console.log('setName: ', this.name);
  };
}

Person.prototype.sex = function () {
  console.log('sex: ');
};

//子实例
function Student(name, age) {
  Person.call(this, name);
  this.age = age;
  this.setAge = function () {
    console.log('setAge: ', this.age);
  };
}

var s = new Student('it8-person', 18); // 可以向父类传参

s.sex(); // 报错, 这里访问不到父类通过原型方式 prototype 添加的属性

s.name = 'it8-s';

  Fn.prototype = obj;
  return new Fn();

  this.name = 'it8';

const p = new Parent();

const s = _create(p);

# 5. 寄生式继承

在原型式继承的基础上，增强对象，返回构造函数

function _create(obj) {

  Fn.getName = function () {
    console.log('getName: ', this.name);
  };

  return Fn;
}

function Parent(name) {
  this.name = name;
  this.friends = [1, 2, 3];
}

const p = new Parent('it8');

const s = _create(p);

console.log(p);

# 6. 寄生组合式继承

结合借用构造函数传递参数和寄生模式实现继承

  • 第一步是创建超类型原型的一个副本
  • 第二步是为创建的副本添加 constructor 属性，从而弥补因重写原型而失去的默认的 constructor 属性
  • 最后一步，将新创建的对象（即副本）赋值给子类型的原型

























 











/* Student: 子元素 Person: 父元素  */
function _create(Student, Person) {
  // 创建对象
  const prototype = Object.create(Person.prototype);
  // 增强对象
  prototype.constructor = Student;
  // 增强对象
  Student.prototype = prototype;
}

//父实例
function Person(name) {
  this.name = name;
  this.play = [1, 2, 3];
  this.setName = function () {
    console.log('setName: ', this.name);
  };
}

Person.prototype.sex = function () {
  console.log('sex: ');
};

//子实例
function Student(name, age) {
  Person.call(this, name);
  this.age = age;
  this.setAge = function () {
    console.log('setAge: ', this.age);
  };
}

_create(Student, Person);

let s1 = new Student('it8', 18);
console.log(s1);

# Promise

Promise 是 ES6 新增的语法，解决了回调地狱的问题。

可以把 Promise 看成一个状态机。初始是 pending 状态，可以通过函数 resolve 和 reject ，将状态转变为 resolved 或者 rejected 状态，状态一旦改变就不能再次变化。

Promise 实现

# API

// 共用
const h1 = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('success-h1');
  }, 1000);
});

const h2 = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('success-h2');
  }, 2000);
});

const h3 = new Promise((resolve, reject) => {
  setTimeout(() => {
    reject('error-h3');
  }, 3000);
});

    • 不带参数传递 — 返回一个新的状态为 resolve 的 promise 对象
